# 文件存放位置分析方案

## 当前目录结构分析

```
src/tqlslib/
  ├── content/
  │   └── MxStats.java
  ├── type/
  │   └── status/
  │       └── ExtentionStatus.java
  ├── worlds/
  │   └── blocks/
  │       ├── distribution/
  │       └── power/
  ├── ExtendedParser.java
  ├── PWIntegration.java
  ├── TQDialogs.java
  ├── TQDrawf.java
  ├── TQLSlib.java
  ├── TQShaders.java
  ├── TQTechTree.java
  ├── TQUtls.java
  └── TranqolIntegration.java
```

## 文件存放位置建议

### 1. 核心集成类
- **TranqolIntegration.java**: 保持在根目录下，与PWIntegration.java同级
  - 理由：作为集成入口点，与其他集成类保持一致的位置

### 2. 图形相关类
- **TQShaders.java**: 建议创建 `graphics/` 目录并放在其中
  - 理由：着色器属于图形相关功能，应该有专门的目录
- **TQDrawf.java**: 建议放在 `graphics/` 目录下
  - 理由：绘图工具与着色器同属图形相关功能，放在同一目录便于管理

### 3. 工具类
- **TQUtls.java**: 建议创建 `util/` 目录并放在其中
  - 理由：通用工具类应该有专门的目录，便于归类和查找

### 4. UI相关类
- **TQDialogs.java**: 建议创建 `ui/` 目录并放在其中
  - 理由：对话框属于UI相关功能，应该有专门的目录

### 5. 内容相关类
- **TQTechTree.java**: 建议放在 `content/` 目录下
  - 理由：技术树属于游戏内容的一部分，与MxStats.java放在同一目录

## 最终目录结构建议

```
src/tqlslib/
  ├── content/
  │   ├── MxStats.java
  │   └── TQTechTree.java
  ├── graphics/
  │   ├── TQDrawf.java
  │   └── TQShaders.java
  ├── type/
  │   └── status/
  │       └── ExtentionStatus.java
  ├── ui/
  │   └── TQDialogs.java
  ├── util/
  │   └── TQUtls.java
  ├── worlds/
  │   └── blocks/
  │       ├── distribution/
  │       └── power/
  ├── ExtendedParser.java
  ├── PWIntegration.java
  ├── TQLSlib.java
  └── TranqolIntegration.java
```

## 建议理由

1. **功能分类明确**：按照功能将文件分类到不同目录，提高代码的可维护性和可读性
2. **结构一致性**：与现有的目录结构保持一致，如content/、type/等
3. **便于查找和管理**：专门的目录使相关功能的文件集中在一起，便于查找和管理
4. **扩展性**：为未来可能添加的类似功能预留了目录结构
